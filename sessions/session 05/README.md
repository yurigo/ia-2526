| Enfoque                                       | Ejemplo lógico                                    | Forma simbólica                        | Traducción a CNF / Resolución                                      | Interpretación                                                                                      |
| --------------------------------------------- | ------------------------------------------------- | -------------------------------------- | ------------------------------------------------------------------ | --------------------------------------------------------------------------------------------------- |
| **Modus Ponens** (“afirmando el antecedente”) | Si llueve → hay charcos. Llueve. ⇒ Hay charcos.   | (P → Q), P ⊨ Q                         | (¬P ∨ Q) ∧ P ⊨ Q → se resuelve eliminando P y queda Q              | En resolución, la inferencia surge al combinar una cláusula que contiene ¬P y otra que contiene P.  |
| **Modus Tollens** (“negando el consecuente”)  | Si llueve → hay charcos. ¬Hay charcos. ⇒ ¬Llueve. | (P → Q), ¬Q ⊨ ¬P                       | (¬P ∨ Q) ∧ ¬Q ⊨ ¬P → resolución sobre Q                            | La contradicción de Q fuerza la negación de P.                                                      |
| **And Elimination**                           | (P ∧ Q) ⇒ P                                       | (P ∧ Q) ⊨ P                            | La CNF mantiene ambas cláusulas: P y Q                             | En CNF ya está descompuesto; cada cláusula se usa individualmente.                                  |
| **Or Introduction**                           | P ⇒ (P ∨ Q)                                       | P ⊨ (P ∨ Q)                            | CNF ya usa disyunciones, por tanto no se necesita regla adicional. |                                                                                                     |
| **Implication Elimination**                   | (P → Q) ⇒ (¬P ∨ Q)                                | Equivalencia lógica                    | Paso obligatorio en conversión a CNF                               | Se usa para eliminar implicaciones antes de resolver.                                               |
| **Double Negation**                           | ¬(¬P) ⇒ P                                         | ¬(¬P) ⊨ P                              | Simplificación directa                                             | Simplificación estructural durante normalización.                                                   |
| **De Morgan**                                 | ¬(P ∧ Q) ⇒ (¬P ∨ ¬Q)                              | Equivalencia lógica                    | Paso obligatorio en CNF                                            | Permite empujar negaciones hasta los literales.                                                     |
| **Resolución (método automático)**            | (P ∨ Q), (¬P ∨ R) ⇒ (Q ∨ R)                       | Reglas sobre literales complementarios | Base del razonamiento algorítmico                                  | Sustituye al conjunto de reglas manuales; combina cláusulas hasta deducir o llegar a contradicción. |
